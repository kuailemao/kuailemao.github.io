1、什么是Redis

百度百科：

Redis（Remote Dictionary Server )，即远程字典服务，是一个开源的使用ANSI C语言编写、支持网络、可基于内存亦可持久化的日志型、Key-Value数据库，并提供多种语言的API。从2010年3月15日起，Redis的开发工作由VMware主持。从2013年5月开始，Redis的开发由Pivotal赞助。

概念：

redis是一款高性能的NOSQL系列的非关系性数据库

1.1、什么是NOSQL

NoSQL，泛指非关系型的数据库。随着互联网web2.0网站的兴起，传统的关系数据库在处理web2.0网站，特别是超大规模和高并发的SNS类型的web2.0纯动态网站已经显得力不从心，出现了很多难以克服的问题，而非关系型的数据库则由于其本身的特点得到了非常迅速的发展。NoSQL数据库的产生就是为了解决大规模数据集合多重数据种类带来的挑战，特别是大数据应用难题。

3、redis的应用场景

​ • 缓存（数据查询、短连接、新闻内容、商品内容等等）
​ • 聊天室的在线好友列表
​ • 任务队列。（秒杀、抢购、12306等等）

​ • 数据过期处理（可以精确到毫秒
​ • 分布式集群架构中的session分离

4、redis的使用

4.1、redis的数据结构

redis存储的是：key,value格式的数据，其中key都是字符串，value有5种不同的数据结构

value的数据结构：
(1) 字符串类型 string
(2) 哈希类型 hash ： map格式
(3) 列表类型 list ： linkedlist格式。支持重复元素
(4) 集合类型 set ： 不允许重复元素
(5) 有序集合类型 sortedset：不允许重复元素，且元素有顺序

4.1.1、字符串类型 string

储存：

set key value

获取：

get key

删除：

del key

4.1.2、哈希类型 hash

储存：

hset key field value

获取：

hget key field**: 获取指定的field对应的值**

hgetall key：获取所有的field和value

删除：

hdel key field

4.1.3、列表类型 list

列表：

可以添加一个元素到列表的头部（左边）或者尾部（右边）

往左添加：

lpush key value

往右添加：

rpush key value

先进先出
获取：

lrange key start end ：范围获取
例如：lrange （key） 0 -1 表示获取全部

删除：

lpop key：删除列表最左边的元素，并将元素返回

rpop key： 删除列表最右边的元素，并将元素返回

4.1.4、集合类型 set

特征：

不允许重复元素

存储：

sadd key value

获取

smembers key：获取set集合中所有元素

删除：

srem key value：删除set集合中的某个元素

4.1.5、有序集合类型 sortedset

特征：

不允许重复元素，且元素有顺序.每个元素都会关联一个double类型的分数。redis正是通过分数来为集合中的成员进行从小到大的排序。

存储：

zadd key score value

score 表示分数 redis 按照分数进行排序

获取

zrange key start end [withscores]

例如：zrange （key） 0 -1 表示获取全部 value
或：zrange （key） 0 -1 withscores 表示获取全部 value 及分数

删除：

zrem key value

4.1.6、通用命令
  1. keys * ：查询所有的键
  2. type key ：获取键（key）对应的value类型
  3. del key ：删除指定的key value
  4. flushall ：删除所有

4.2、持久化

redis是一个内存数据库，当redis服务器重启，或者电脑重启，数据会丢失，我们可以将redis内存中的数据持久化保存到硬盘的文件中。

持久化机制：

redis 有两种持久化方式

  1. ROB
  2. AOF
4.2.1、ROB

默认方式，不需要进行配置，默认就使用这种机制

  1. 在一定的间隔时间中，检测key的变化情况，然后持久化数据
    修改配置文件：

可以通过编辑 redis.windwos.conf 文件里面的 save 参数，得到自己想要 redis 的保存的时间

如验证码24h之后失效

  1. 之后重启服务器
4.2.2、AOF

日志记录的方式，可以记录每一条命令的操作。可以每一次命令操作后，持久化数据

  1. 编辑redis.windwos.conf文件

    把 appendonly no（关闭aof） 改成 appendonly yes （开启aof）

默认**appendfsync **everysec 表示每隔一秒进行一次持久化

appendfsync 可修改为 ：

  • appendfsync always ： 每一次操作都进行持久化
  • appendfsync no ： 不进行持久化

5. Java客户端 Jedis

  • Jedis: 一款java操作redis数据库的工具.
  • 使用步骤：
    1. 下载jedis的jar包
    2. 使用
      //1. 获取连接
      Jedis jedis = new Jedis(“localhost”,6379);
      //2. 操作
      jedis.set(“username”,”zhangsan”);
      //3. 关闭连接
      jedis.close();

注意：

Jedis jedis = new Jedis(); 括号里面没有参数的时候默认就是本地地址 127.0.0.1 6379
